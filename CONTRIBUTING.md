# Contributing to Predoc Coding Sample

Thank you for your interest in this portfolio repository!

## Purpose

This repository serves as a coding portfolio for Predoc position applications. It demonstrates:
- Programming proficiency in Python and R
- Statistical and econometric analysis skills
- Clean, well-documented code
- Research-oriented programming practices

## Code Organization

The repository is organized into four main sections:
- `data-analysis/`: Data cleaning, exploration, and visualization
- `statistical-analysis/`: Statistical methods and econometric models
- `research-projects/`: Research-oriented projects and replications
- `utilities/`: Reusable helper functions and tools

## Code Style

### Python
- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Include docstrings for all functions
- Add inline comments for complex logic

### R
- Follow tidyverse style guide
- Use roxygen2 documentation format
- Keep functions modular and reusable
- Include examples in documentation

## Documentation

- Each directory contains a README.md explaining its contents
- Code files include header comments with author and date
- Functions have detailed docstrings/documentation
- Complex algorithms include explanatory comments

## Testing

While this is a portfolio repository without formal tests:
- All scripts should run without errors
- Functions should handle edge cases gracefully
- Results should be reproducible with set random seeds
